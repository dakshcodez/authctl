# Phase 4 — Session Store

The session store is a convenience cache on disk. It lets the CLI know who is logged in between commands without talking to the database on every keystroke. It is **not** the source of truth for authentication — the database is.

## Files

```
internal/session/
  store.go       — StoredSession, Store interface, fileStore
  store_test.go  — tests for save/load/expiry/corruption
```

## What it is and what it is not

The CLI is an interactive shell. Between commands, the shell needs to know:

- Is there a logged-in user?
- What is their username (for the prompt)?
- What is their session token (for service calls)?

The session store answers these questions without a database round-trip. It writes a small JSON file to `~/.authctl/session` after login and reads it back at startup and before each command.

It is not an auth decision point. The service layer (`ValidateSession`) re-validates the token against the database whenever the result actually matters (e.g., `whoami`, `mfa` commands). The file store is purely a way to avoid prompting for a password on every command.

## `StoredSession`

```go
type StoredSession struct {
    Token     string    `json:"token"`
    Username  string    `json:"username"`
    ExpiresAt time.Time `json:"expires_at"`
}

func (s *StoredSession) IsExpired() bool {
    return time.Now().After(s.ExpiresAt)
}
```

Only what the CLI needs is stored. The user ID is not stored here — the CLI always fetches the full user from the database when it needs to make an auth decision.

`Token` holds the **plaintext** session token. This is the value generated by `generateToken()` in the service layer. The hashed version lives in the database. The store presents the plaintext token back to the service so `SessionRepository.GetByToken` can hash it and look it up.

## `Store` interface

```go
type Store interface {
    Save(s *StoredSession) error
    Load() (*StoredSession, error)
    Clear() error
}
```

The CLI depends on this interface, not the concrete `fileStore`. Tests inject a fake store without touching the filesystem.

## File path

```go
func DefaultPath() (string, error) {
    home, err := os.UserHomeDir()
    // ...
    return filepath.Join(home, ".authctl", "session"), nil
}
```

`~/.authctl/session` is the default. In Docker, the `VOLUME` is mounted at `/home/authctl/.authctl`, so the file persists across container restarts in the named volume.

## Security properties

**Directory permissions: 0700**
`Save` creates the `~/.authctl/` directory with mode `0700` if it does not exist. Other users on the same machine cannot read or list the directory.

```go
os.MkdirAll(filepath.Dir(f.path), 0700)
```

**File permissions: 0600**
The session file is written with mode `0600`. Only the owner can read it.

```go
os.WriteFile(f.path, data, 0600)
```

**Expiry enforced on load**
`Load` checks `IsExpired()` after unmarshalling. If the session is expired, it immediately calls `Clear()` and returns `ErrNoSession`. An expired session file cannot be used, even if somehow read directly.

**Corrupted file treated as no session**
If the file exists but cannot be unmarshalled (truncated, manually edited, wrong format), `Load` returns `ErrNoSession` rather than an error. The CLI treats this the same as "not logged in". This avoids a crash loop on a corrupted file.

## `ErrNoSession`

```go
var ErrNoSession = errors.New("no active session")
```

The sentinel for "no session file" and "expired session". Callers check `errors.Is(err, session.ErrNoSession)` and display "Not logged in." rather than an error.

## CLI integration

Login writes the session:
```go
h.store.Save(&session.StoredSession{
    Token:     result.Session.Token,
    Username:  username,
    ExpiresAt: result.Session.ExpiresAt,
})
```

Startup reads it to set the prompt:
```go
// handler.go: Init()
if stored, err := h.store.Load(); err == nil {
    h.prompter.SetPrompt(UserPrompt(stored.Username))
}
```

Logout clears it:
```go
h.store.Clear()
h.prompter.SetPrompt(DefaultPrompt())
```

Session validation failure (expired token in DB) also clears it:
```go
if err := h.auth.ValidateSession(...); err != nil {
    h.store.Clear()
    h.prompter.SetPrompt(DefaultPrompt())
    warn(h.out, "Session expired — please login again.")
}
```

The prompt and the store are always kept in sync. There is no path where the prompt shows a username but the store is empty, or vice versa.
